How a Browser Requests and Displays a Web Page

  1. URL entered or link clicked: The browser parses the URL (scheme, host, path, query).
  2. DNS lookup: The browser resolves the domain name to an IP address via DNS.
  3. TCP connection & TLS handshake: The browser opens a TCP connection to the server and performs a TLS handshake for HTTPS.
  4. HTTP request: The browser sends an HTTP request (for example, GET /index.html) including headers (User-Agent, Accept, Cookies).
  5. Server processing: The web server receives the request, routes it to the appropriate handler, which may query databases or call other services, and prepares a response (HTML, JSON, etc.).
  6. HTTP response: The server sends back the response with status code (200 OK, 404, etc.) and response headers (Content-Type, Cache-Control).
  7. Browser parsing: The browser parses the HTML. While parsing, it builds a DOM (Document Object Model). It discovers CSS and JS resources and requests them as needed.
  8. CSS & JS handling: The browser fetches CSS (builds CSSOM) and JS (executes scripts). Render-blocking scripts may pause parsing until executed.
  9. Layout and paint: The browser combines DOM and CSSOM into a render tree, computes layout, and paints pixels to the screen.
  10. User interaction: After rendering, event listeners and further dynamic updates (DOM changes) can modify the page.
