Basic Web Architecture Flow

[Client / Browser]
     |
     | -- (1) HTTP Request (e.g., GET /products)
     v
[Web Server / Application Server]
     |  (Routes request, runs business logic)
     | -- (2) API Call / DB Query --> [Database]
     |                         <-- (3) Data Response
     v
[API Response]
     |
     | -- (4) HTTP Response (JSON/HTML)
     v
[Client renders response]

Components:
 - Client: Browser or mobile app that sends requests
 - Web Server / App Server: Receives requests, runs application code, exposes APIs
 - Database: Persistent storage (SQL/NoSQL)
 - APIs: Interfaces (REST/GraphQL) used for communication between client and server components
